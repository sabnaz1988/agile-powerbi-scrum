# Project Outline
*Created based on alignment outcomes during [Team Meeting 01](../sprint-notes/team-meeting-01.md).*

## Name of the Project

DataLens Streaming

## Project Purpose

To facilitate the company's transition from a free, ad-supported platform to a subscription-based service by identifying which film genres to prioritize for acquisition and promotion.

## Goals and Objectives of the Project

- **Expected outcomes:** Actionable insights into user preferences to ensure the platform buys films that existing users appreciate while simultaneously attracting new subscribers to support global growth.
- **What the team will produce:** A data-driven dashboard featuring clear, easily interpretable visualizations for stakeholders.

## Defining the Project Needs

- **Data to be used:** Objective data encompassing user age, user location, subscription type, film genres, and individual movie performance metrics.
- **Specific questions:** What types of films do different groups of users enjoy based on their demographics and subscription status? What are the most successful genres and movies for each specific audience?
- **Time or resource limitations:** The timeline is extremely tight, with the platform revamp launching in a few weeks. The first version of the project proposal must be delivered by July 1st.

## Project Life Cycle

### Planning

- **Expected outcomes:** A finalized project scope and timeline to meet the strict impending deadline.
- **What the team will produce:** The initial project proposal and a clear roadmap to deliver the dashboard by July 1st.

### Analysis

- Review available datasets regarding user demographics (age, location), subscription tiers, and viewing history.
- Identify any immediate data gaps or technical limitations that could impact the tight turnaround.

### Design

- Plan the logical relationships between the demographic tables and the movie/genre performance tables.
- Wireframe the dashboard layout to ensure it prioritizes clear, high-level visualizations tailored for non-technical stakeholders.

### Implementation

- Clean and merge the user and film datasets.
- Conduct the core analysis to segment the most successful genres and specific movies by age, location, and subscription type.

### Testing & Integration

- Validate the generated insights to ensure they are strictly based on objective data.
- Test the dashboard visualizations to confirm they directly answer the client's key questions about content prioritization.

### Presentation

- Deliver the initial proposal and first version of the dashboard by July 1st.
- Highlight the key findings regarding audience preferences and provide objective recommendations for future film acquisitions.
- Note limitations encountered with data.

### Maintenance and Follow-Up

- Establish a pipeline to update the dashboard with fresh streaming data once the revamped platform officially launches.
- Plan for future analysis to track how the newly acquired content performs in supporting the company's ongoing global growth ambitions.
